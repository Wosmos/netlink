SECTION.03
Frequently Questioned Answers


  ▸ Is this a real framework?
Define "real." In a post-framework world, the most real framework is the one that doesn't pretend to be something it's not. VOID is authentically nothing, and we think that's beautiful.
  ▸ How does it compare to React?
React re-renders. VOID doesn't render in the first place. You can't have render performance issues if you never render. We call this "Zen Rendering" — the art of displaying nothing, perfectly.
  ▸ What about TypeScript support?
We support every type: string, number, boolean, undefined, null, symbol, bigint, and most importantly — void. Especially void.
  ▸ Can I use it in production?
VOID is already running in production on every website that doesn't use a framework. So technically, most of the internet is already built on VOID. You just didn't know it.
